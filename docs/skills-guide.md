# Skills Guide

Skills are self-contained bundles of instructions + tools that a Hipp0 agent
can load on demand. The format matches [agentskills.io](https://agentskills.io):
`SKILL.md` + `manifest.json` + optional `tools.ts`.

## Layout

```
skills/
└── my-skill/
    ├── SKILL.md          # natural-language instructions
    ├── manifest.json     # metadata + permissions
    └── tools.ts          # optional: additional tools
```

## manifest.json

```json
{
  "name": "my-skill",
  "version": "0.1.0",
  "description": "What this skill does.",
  "trigger": "keyword patterns or regex",
  "permissions": ["fs.read", "net.fetch"],
  "tags": ["example"]
}
```

Validation: `SkillManifestSchema` (Zod) — fields are enforced at load time,
not at registry insertion.

## Loader precedence

Workspace > user global > built-in:

1. `./skills/` in the current project
2. `~/.hipp0/skills/`
3. Packages that export `skillsPath` (built-in skills shipped with core)

Dot-prefixed and `_`-prefixed directories (e.g. `_template/`) are skipped.

## Skill discovery

The agent loads a skill when its trigger pattern matches the current user
turn or when it's explicitly requested via the `skill` CLI:

```bash
hipp0 skill list
hipp0 skill search "web research"
hipp0 skill audit           # validate every manifest
hipp0 skill create my-new-skill
```

## Self-creation

When an agent completes a task that involved 5+ tool calls and no existing
skill matches the shape, the self-learning loop (see
`packages/memory/src/learning/skills.ts`) can auto-create one. The new
skill is stored in the DB with `autoGenerated = 1` and an initial
`trigger_pattern` derived from the session.

## Self-improvement

When a skill fails or deviates from expected output, the loop generates a
diff + new version (`skills.parentVersionId → skillImprovements`). Track
`successRate` over time to identify which skills need attention.

## Testing a skill

```bash
hipp0 skill test <name>     # placeholder in Phase 8; full test runner lands in a later phase
```

Until the test runner lands, import the skill's tools directly and invoke
them through the usual Vitest pattern.

## Name rules

Lowercase slug only: `/^[a-z0-9_-]+$/`. Names become directory names,
registry keys, and command-line arguments.
